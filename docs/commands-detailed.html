Git Commands

These commands work on your local git repository and analyze commit history with focus on timestamps and author information.

tv log [file]

Show commit history with color-coded age timestamps. This command displays all commits in the repository or for a specific file, with timestamps color-coded by age to quickly visualize the lifecycle of your codebase.

Usage: tv log [options] [file]

Options

General Options

-r, --repo <path>

Path to the git repository (default: .)

-n, --max-count <n>

Limit the number of commits to display

Examples

# Show all commits in the repository
tv log

# Show commits for a specific file
tv log src/cli.ts

# Show only the last 10 commits
tv log -n 10

# Show commits for a specific repository
tv log --repo /path/to/repo

tv summary [path]

Show per-file timestamp statistics including the newest change, first commit date, total commit count, and a list of authors. This gives you a quick overview of which files are actively maintained and their history.

Usage: tv summary [options] [path]

Options

General Options

-r, --repo <path>

Path to the git repository (default: .)

Displayed Columns

Column Description
File Path to the tracked file
Newest Date of the most recent commit
First Date of the initial commit
Commits Total number of commits touching this file
Authors List of authors who have modified this file

Examples

# Show statistics for all files
tv summary

# Show statistics for a specific directory
tv summary src/

# Show statistics for a specific file
tv summary src/cli.ts

# Show statistics for a different repository
tv summary --repo /path/to/repo

tv annotate <file>

Show per-line blame with age-colored timestamps, similar to git blame. Each line displays the timestamp of the last commit that touched it, color-coded by age to quickly identify recent changes and legacy code.

Usage: tv annotate [options] <file>

Options

General Options

-r, --repo <path>

Path to the git repository (default: .)

--no-author

Omit the author column from the output

Displayed Columns

Column Description
Line Number The line number in the file
Timestamp When the line was last modified (color-coded by age)
Author Who made the last modification (optional)
Code The actual line content

Examples

# Annotate a file with timestamps and authors
tv annotate src/cli.ts

# Annotate without showing authors
tv annotate src/cli.ts --no-author

# Annotate a file in a different repository
tv annotate --repo /path/to/repo src/cli.ts

GitHub Commands

These commands interact with the GitHub API to extract timestamped events and workflow information. You'll need a GitHub personal access token to use these commands.

Authentication

All GitHub commands require authentication. Provide your token using one of these methods:

# Option 1: Set as environment variable
export GITHUB_TOKEN=your_token_here

# Option 2: Pass as command-line flag
tv gh events owner/repo --token your_token_here

tv gh events <owner/repo>

Show all timestamped events from the GitHub API. This includes comments, code reviews, pull request reviews, commit comments, and workflow runs. Filter by event type to focus on specific kinds of activity.

Usage: tv gh events [options] <owner/repo>

Options

General Options

-t, --token <token>

GitHub personal access token (or set GITHUB_TOKEN environment variable)

-k, --kind <kinds>

Comma-separated event kinds to include. Available kinds:

  • issue_comment - Comments on issues
  • pr_review_comment - Review comments on pull requests
  • pr_review - Pull request reviews
  • commit_comment - Comments on commits
  • workflow_run - Workflow execution events

Examples

# Show all GitHub events for a repository
tv gh events JakeDot/temporal-vortex

# Show only pull request reviews
tv gh events JakeDot/temporal-vortex --kind pr_review

# Show issue comments and pull request reviews
tv gh events JakeDot/temporal-vortex --kind issue_comment,pr_review

# Use a specific token
tv gh events JakeDot/temporal-vortex --token gh_1234567890abcdef

tv gh workflows <owner/repo>

Show GitHub Actions workflow runs with agent start and end timestamps. This command displays the workflow name, status, duration, and timing information to help you track CI/CD pipeline performance and identify bottlenecks.

Usage: tv gh workflows [options] <owner/repo>

Options

General Options

-t, --token <token>

GitHub personal access token (or set GITHUB_TOKEN environment variable)

Output Information

Field Description
Timestamp Range Start and end time of the workflow run
Status Success, failure, or other status of the run
Workflow Name Name of the GitHub Actions workflow
Trigger Event What triggered the workflow (e.g., push, pull_request)
Duration Total time the workflow took to complete

Examples

# Show all workflow runs
tv gh workflows JakeDot/temporal-vortex

# Show workflow runs with a specific token
tv gh workflows JakeDot/temporal-vortex --token gh_1234567890abcdef

# Pipe to other tools for analysis
tv gh workflows JakeDot/temporal-vortex | grep "FAILURE"

Getting Help

Use the --help flag to display command help information at any time.

Examples

# Show general help
tv --help

# Show help for a specific command
tv log --help
tv summary --help
tv annotate --help
tv gh --help
tv gh events --help
tv gh workflows --help
